# 스테이트 칼리지 한인교회 웹사이트 문서

이 디렉토리는 스테이트 칼리지 한인교회 웹사이트의 디자인 시스템과 개발 가이드라인을 포함합니다.

## 문서 구조

### 📋 [design-system.md](./design-system.md)
웹사이트의 전반적인 디자인 철학과 시스템을 정의합니다.

**주요 내용:**
- 디자인 철학 (미니멀리즘, 모던 디자인, 접근성)
- 컬러 시스템 및 사용 가이드라인
- 타이포그래피 체계
- 레이아웃 및 간격 시스템
- 브랜드 가이드라인

### 🎨 [ui-patterns.md](./ui-patterns.md)
실제 개발에서 사용되는 UI 패턴과 코드 예제를 제공합니다.

**주요 내용:**
- 페이지 레이아웃 패턴
- 컴포넌트 패턴 (카드, 버튼, 폼 등)
- 반응형 디자인 패턴
- 애니메이션 및 상호작용 패턴
- 접근성 및 성능 최적화 패턴

## 디자인 컨셉 요약

### 🎯 핵심 가치
1. **미니멀리즘**: 불필요한 요소 제거, 콘텐츠 중심
2. **모던함**: 깔끔한 타이포그래피, 기하학적 형태
3. **접근성**: 모든 사용자를 위한 포용적 디자인

### 🎨 시각적 특징
- **블랙 & 화이트** 중심의 고대비 색상
- **둥근 모서리**와 **부드러운 그라데이션**
- **충분한 여백**과 **명확한 위계구조**
- **기하학적 도트** 요소로 브랜드 정체성 강화

### 📱 사용자 경험
- **모바일 우선** 반응형 디자인
- **직관적 네비게이션** 구조
- **빠른 로딩** 속도 최적화
- **명확한 액션** 버튼과 피드백

## 기술 스택

### Frontend
- **Next.js 13+**: React 기반 풀스택 프레임워크
- **TypeScript**: 타입 안전성
- **Tailwind CSS**: 유틸리티 기반 스타일링
- **Lucide React**: 아이콘 라이브러리

### Backend & Database
- **Firebase Firestore**: NoSQL 데이터베이스
- **Firebase Admin SDK**: 서버사이드 인증/권한
- **Vercel**: 배포 및 호스팅

### 개발 도구
- **ESLint**: 코드 품질 관리
- **Prettier**: 코드 포맷팅
- **Git**: 버전 관리

## 컴포넌트 아키텍처

### 레이아웃 컴포넌트
```
Layout/
├── Header (네비게이션)
├── Main (페이지 내용)
└── Footer (하단 정보)
```

### 페이지 구조
```
pages/
├── index.tsx (홈페이지)
├── about/ (교회 소개)
├── worship/ (예배 안내)
├── ministries/ (사역)
├── education/ (교육)
├── news/ (교회 소식)
└── contact/ (연락처)
```

### 공통 컴포넌트
- **Cards**: 정보 표시용 카드 컴포넌트
- **Buttons**: 다양한 스타일의 버튼
- **Forms**: 입력 폼 및 검증
- **Navigation**: 탭, 브레드크럼 등

## 개발 가이드라인

### 코딩 컨벤션
- **컴포넌트명**: PascalCase
- **파일명**: kebab-case
- **CSS 클래스**: Tailwind 유틸리티 우선
- **폰트**: `font-korean` 클래스 사용

### 반응형 설계
```css
/* 모바일 우선 */
.component {
  @apply text-sm p-4;
}

/* 태블릿 */
@screen md {
  .component {
    @apply text-base p-6;
  }
}

/* 데스크톱 */
@screen lg {
  .component {
    @apply text-lg p-8;
  }
}
```

### 성능 최적화
- **이미지**: Next.js Image 컴포넌트 사용
- **폰트**: 서브셋 로딩
- **코드 분할**: 동적 임포트 활용
- **캐싱**: ISR(Incremental Static Regeneration) 활용
